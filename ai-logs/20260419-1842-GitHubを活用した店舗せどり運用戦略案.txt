# AIログ：GitHubを活用した店舗せどり運用戦略の構築（2026-04-19）

## 1. 運用の全体コンセプト
「思考とプロセス」は **GitHub** で管理し、「数値と結果」は **Googleスプレッドシート** で管理する。これにより、なぜ利益が出たのか（あるいは損失が出たのか）を言語化し、仕入れ判断の精度を継続的に向上させるシステムを構築する。

## 2. GitHub リポジトリ構成案
`style-guidelines.md` に準拠したディレクトリ構成。

```text

  docs/

    manuals/

    analysis/

  ai-logs/
    2026/

  README.md                 # 運用フロー全体図とスプレッドシートへのリンク
```

## 3. 仕入れ判断・戦略立案の標準フォーマット
`shop-arbitrage.md` に基づく、仕入れ時に Issue へ入力すべき項目。

### 3.1. 入力データ（商品情報）
- 商品名：[記入]
- 仕入れ価格：[価格]円
- Amazon/メルカリ現在相場：[価格]円
- 利益計算（手数料・送料引後）：約[金額]円
- 回転率（ランキング/売れ行き）：[良い/普通/悪い]
- コンディション：[新品/中古良品/難あり]

### 3.2. 戦略分析ステップ
1. **妥当性の検証（技法27引用）**: 資金回収の速さと赤字リスクから1〜10点で評価。
2. **リスクの検証（技法30引用）**: 出品制限、真贋調査、季節性、競合増の予兆を確認。
3. **魅力的な見せ方（技法36引用）**: 併売時の訴求ポイント3行と言語化。

## 4. 作業手順（ワークフロー）

1. **リサーチ・仕入れ（GitHub Issues）**
   - 店頭で候補を見つけたらスマホアプリから Issue を作成。
   - テンプレートに沿ってセルフチェックを行い、仕入れ可否を判断。
2. **数値記録（スプレッドシート）**
   - 帰宅後、仕入値をシートに入力。備考欄に GitHub Issue 番号（#123 等）を付与。
3. **進捗管理（GitHub Projects）**
   - Issue を「仕入れ済」→「FBA配送中」→「販売中」とカード移動させて可視化。
4. **月次改善（GitHub Docs）**
   - 月末、スプレッドシートで利益の低かった商品を特定。
   - 該当 Issue を読み返し、**技法40（阻害要因の特定）**を用いて判断のズレを分析。
   - 分析結果を元に、翌月の仕入れ基準（マニュアル）を更新する。

## 5. 本運用のメリット
- **再現性の向上**: 感覚ではなく、テンプレートに基づいた論理的な仕入れが可能になる。
- **資産化**: 過去の失敗と成功がすべて GitHub に蓄積され、自分専用の攻略本になる。
- **拡張性**: 将来的に納品代行などの外注化を行う際、マニュアルの共有が容易。
